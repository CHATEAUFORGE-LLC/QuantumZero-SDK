# QuantumZero Rust SDK

A Rust library for the QuantumZero Identity Platform. This SDK provides W3C-compliant cryptographic operations for issuers and service integrators, including Ed25519 key generation, signing, and verification to support Decentralized Identity (DID) workflows.

## Features

- **W3C Compliance:** Generates `Ed25519` key pairs compatible with `did:key` standards
- **Cryptographic Primitives:**
  - **Signing:** Ed25519 (Edwards-curve Digital Signature Algorithm)
  - **Hashing:** SHA-256 (for content addressing and ZKP inputs)
  - **Hardware Keys:** P-256 (NIST) signature verification for Android StrongBox integration
- **Issuer Integration:** Designed for issuers to integrate with the QuantumZero service
- **Performance:** Native Rust performance with zero-cost abstractions
- **Comprehensive Testing:** 26+ unit tests covering edge cases and security scenarios

## Target Use Cases

This SDK is designed for:
- **Issuers:** Organizations that need to register with QuantumZero and issue verifiable credentials
- **Service Integrators:** Backend services that need to generate and verify DIDs
- **Rust Applications:** Any Rust application requiring QuantumZero-compatible cryptographic operations

## Prerequisites

- Rust 1.70.0 or higher (2021 edition)

## Installation

Add the dependency to your `Cargo.toml`:

```toml
[dependencies]
quantumzero_rust_sdk = { git = "https://github.com/CHATEAUFORGE-LLC/QuantumZero-SDK", branch = "main" }
```

Or for local development:

```toml
[dependencies]
quantumzero_rust_sdk = { path = "../QuantumZero-SDK/quantumzero_rust_sdk" }
```

## Usage

### Generating a Key Pair

```rust
use quantumzero_rust_sdk::CryptoCore;

fn main() {
    let crypto = CryptoCore::new();
    
    // Generate a secure, W3C-compliant key pair
    let key_pair = crypto.generate_key_pair();
    
    println!("Public key: {:?}", key_pair.verifying_key.to_bytes());
}
```

### Signing Data

```rust
use quantumzero_rust_sdk::CryptoCore;

fn main() {
    let crypto = CryptoCore::new();
    let key_pair = crypto.generate_key_pair();
    
    let data = b"Important message to sign";
    let signature = crypto.sign_data(data, &key_pair);
    
    println!("Signature: {:?}", signature.to_bytes());
}
```

### Verifying Signatures

```rust
use quantumzero_rust_sdk::CryptoCore;

fn main() {
    let crypto = CryptoCore::new();
    let key_pair = crypto.generate_key_pair();
    
    let data = b"Important message to sign";
    let signature = crypto.sign_data(data, &key_pair);
    
    // Verify the signature
    let is_valid = crypto.verify_signature(
        data,
        &signature,
        &key_pair.verifying_key,
    );
    
    println!("Signature valid: {}", is_valid);
}
```

### Hashing Data

```rust
use quantumzero_rust_sdk::CryptoCore;

fn main() {
    let crypto = CryptoCore::new();
    let data = b"Data to hash";
    let hash = crypto.hash_data(data);
    
    println!("SHA-256 Hash: {:?}", hex::encode(hash));
}
```

### Verifying Hardware Signatures (P-256)

For verifying signatures from hardware-backed keys (e.g., Android StrongBox):

```rust
use quantumzero_rust_sdk::CryptoCore;

fn main() {
    let crypto = CryptoCore::new();
    
    // P-256 public key (SEC1 encoded, 33 or 65 bytes)
    let public_key_bytes = vec![/* ... */];
    
    // Signature from hardware key (ASN.1 DER format)
    let signature_bytes = vec![/* ... */];
    
    let data = b"Data signed by hardware key";
    
    let is_valid = crypto.verify_hardware_signature(
        data,
        &signature_bytes,
        &public_key_bytes,
    );
    
    println!("Hardware signature valid: {}", is_valid);
}
```

## Interoperability

This Rust SDK is designed to be **fully interoperable** with the Dart SDK:
- Both use Ed25519 for signing
- Both use SHA-256 for hashing
- Both follow W3C DID standards
- Signatures generated by one can be verified by the other

## Integration with QuantumZero Server

For issuers integrating with the QuantumZero Rust server, add this SDK as a dependency:

```toml
[dependencies]
quantumzero_rust_sdk = { path = "../QuantumZero-SDK/quantumzero_rust_sdk" }
```

Then use it in your issuer registration and credential issuance workflows:

```rust
use quantumzero_rust_sdk::CryptoCore;

// Generate issuer DID
let crypto = CryptoCore::new();
let issuer_key_pair = crypto.generate_key_pair();

// Sign credential
let credential_data = serde_json::to_vec(&credential)?;
let signature = crypto.sign_data(&credential_data, &issuer_key_pair);
```
